Hello agents学习chapter1

今天学习的是 Datawhale《Hello Agents》的第一章《初识智能体》。这一章像是给 Agent 入门打地基：先回答“智能体是什么”，再解释它如何感知环境、做出决策、调用工具并完成目标。

我对 Agent 的第一层理解

这一章给出的核心定义是：智能体是一个能够感知环境，并通过行动影响环境、完成目标的实体。它不只是一个会回答问题的模型，而是一个处在“感知-思考-行动-反馈”循环里的系统。

如果把大模型看作“大脑”，那么 Agent 更像是带着大脑、眼睛、手和任务目标的人。它需要通过传感器获得信息，通过执行器做出动作，并根据结果继续调整下一步。

传统智能体到 LLM 智能体

文章先回顾了传统智能体的几种形态：简单反射智能体、基于模型的智能体、基于目标的智能体、基于效用的智能体，以及能够从经验中改进的学习型智能体。

我觉得这里最重要的不是背分类，而是看懂它们的演进路线：从“看到什么就按规则反应”，到“维护一个内部世界模型”，再到“为了目标规划行动”，最后到“通过反馈不断学习”。这条线索能帮助我理解为什么今天的 LLM Agent 不只是聊天机器人。

LLM 驱动的智能体把大语言模型作为核心推理引擎。它可以理解自然语言目标，拆解任务，选择工具，读取工具返回的结果，再继续下一轮判断。比如“帮我规划一次旅行”不再只是生成一段建议，而可以拆成查天气、查景点、比酒店、调整预算等一系列动作。

三个分类角度

这一章还提供了几个观察智能体的角度：

  • 按内部决策架构看：反射式、模型式、目标式、效用式、学习型。
  • 按反应时间看：反应式、规划式、混合式。
  • 按知识表示看：符号主义、亚符号主义、神经符号主义。

我最有感觉的是“反应式”和“规划式”的区别。反应式像安全气囊，必须立刻行动；规划式像下棋，需要先推演。现实里的 Agent 往往是混合式：先想一步，再做一步，观察反馈后继续想。

PEAS：描述任务环境的方法

文章用 PEAS 模型来描述智能体的任务环境。PEAS 分别是 Performance、Environment、Actuators、Sensors，也就是性能度量、环境、执行器和传感器。

这个模型让我意识到，设计 Agent 不能只问“让它做什么”，还要问：它怎么知道自己做得好不好？它面对的环境是什么？它能调用哪些工具？它能看到哪些信息？

Thought-Action-Observation 循环

本章实践部分最关键的是 Thought-Action-Observation 循环。它可以理解为：

  • Thought：先分析当前情况，想清楚下一步。
  • Action：选择一个工具或动作去执行。
  • Observation：读取执行结果，作为下一轮思考的依据。

这个循环解释了为什么 Agent 能处理多步骤任务。它不是一次性给答案，而是在每一步行动后吸收新信息，让任务逐渐逼近目标。

Workflow 和 Agent 的区别

这一章最后讲了 Workflow 和 Agent 的差异。Workflow 是预先写好的固定流程，适合规则稳定、边界清晰的任务。Agent 则更像目标导向的执行者，适合开放、模糊、需要动态判断的任务。

我的理解是：Workflow 像流水线，Agent 像实习生。流水线稳定、可控、便宜，但只能按设定流程走；实习生能理解目标并临场判断，但也需要边界、检查和监督。

收获

学完这一章，对 Agent 的理解从“会调用工具的大模型”变成了“围绕目标运行的感知-决策-行动系统”。工具调用只是表面，更核心的是它能不能在环境反馈中持续调整行动。
